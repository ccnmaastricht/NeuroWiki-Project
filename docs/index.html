Why NeuroWiki

Built for how neuroscientists actually work

Not a reference manager. Not a vector database. A structured, agent-maintained synthesis that compounds over time.

📄→🧠

Papers become knowledge

Drop a PDF in raw/ and run the agent. Ten papers on V1 layer 4 synthesize into one REG_v1-layer4 page — not ten summaries.

†

Epistemic status always visible

Every claim is either (@Key) — verified from a PDF you hold — or (@Key†) — reconstructed from another paper's citation. You always know how much to trust what you're reading.

⚑

Conflicts surfaced, not silenced

When papers contradict each other the agent flags the conflict for human review rather than silently choosing a side. Science is messy; the wiki reflects that.

🔗

Agent-navigable structure

OKF-compatible YAML frontmatter and a root wiki/index.md let agents navigate by progressive disclosure — no loading everything at once.

✓

Humans stay in the loop

Every session ends with a human verification checklist. Claims are spot-checked against source PDFs. No page reaches a manuscript without sign-off.

🌐

Scales to the lab

Project wikis submit to a lab wiki via pull request. Evidence from all projects in a lab is synthesized into a shared lab-level knowledge base, with each project credited in every page it contributes to.

Architecture

Three layers, one knowledge system

Start with a single project. Merge into a lab-level wiki.

NeuroWiki

Project level — this repo

One copy per research line. PDFs go in raw/; the agent synthesizes them into concept pages. Verified by humans after every session.

Lab wiki

Lab level — separate repo

Merges project wikis from across a lab into a unified synthesis. The lab wiki runs a merge script and LLM agent workflow to synthesize overlapping pages, not concatenate them.

Citation model

Two tiers. Always visible.

The dagger marks the difference between what you've verified and what you're trusting someone else to have verified.

## Empirical Basis

Place cells in CA1 fire reliably in circumscribed regions of an environment,
with different cells tiling space (@OKeefe1971).
When the animal enters a novel environment, a largely uncorrelated population
becomes active — global remapping (@Muller1987†).

## Key Parameters

Population vector correlations drop to near zero across environments (@Leutgeb2005†).
Approximately 30–40% of CA1 pyramidal cells are active per environment (@Wilson1993).
(@Key)  PDF in raw/ → primary.bib · verified (@Key†) reconstructed from citation → secondary.bib · unverified
How it works

Set up in minutes. Compounds over years.

1

Use the template and run setup

Click Use this template on GitHub, clone your new repo, fill in your project details in AGENT.md, and run bash setup.sh "My Project". Done.

2

Add PDFs and run ingestion

Drop papers into raw/. Paste the ingestion prompt from QUICKSTART.md into your agent of choice. The agent reads, synthesizes, and updates the wiki.

3

Verify and sign off

A lab member spot-checks claims against source PDFs using the checklist in VERIFICATION.md and signs the session log.

4

Refine periodically

Run REFINE_A (structure), REFINE_B (depth audit), and REFINE_C (cross-links) to keep the wiki consistent, detailed, and navigable.

5

Contribute to your lab wiki

Run validate.py, then submit via pull request to your lab wiki repository. A lab maintainer reviews the merge report and approves. Your evidence enriches the shared lab-level knowledge base.
